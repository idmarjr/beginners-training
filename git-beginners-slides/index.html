Where am I?


							$ pwd

$ pwd stands for “Print working directory"

It will show you in witch folder you are right now.

What do I have here?


							$ ls

$ ls is an abreviation of "list".

It will list existent files and directories in the folder that you are currently in.

What do I have here?


							$ ls -1

By using the argument “-1” in combination with the command “ls" will list files vertically. This is just a different way to see the same information.

There are a lot of different arguments that you can use if you want to visualize it in different ways.

If you use “ls -la“ for example, it will bring you the same list with more detailed information for each file/folder.

More info: https://explainshell.com/explain/1/ls

Clear the view


							$ clear

The “clear" command will clear the view of your bash window.

Be notice that it not erase the previous information. If you scroll up, all history still there.

Change directory


							$ cd folder_name

$ cd stands for “Change directory”.

Notice that the folder that you intend to navigate in, need to be inside of the actual fonder. To see a list all available folders to navigate you can hit TAB two times after type “cd"

Be aware that you can also specify a path of folders. For example: $cd Developer/booking/main

A step back


							$ cd ..

In order to navigate back in the tree you can use “..”. It will make you return one level in the folder tree.

You can also go back more levels at once: $ cd ../../

Make a directory


							$ mkdir folder_name

$ mkdir stands for Make directory.

Create a file


							$ touch file_name.txt

$ touch followed by a name + extension will create a empty file of that specific kind.

Open a file

Mac


							$ open file_name.txt

$ open followed by a name + extension will open that file using the default editor for that kind of file on Mac OS X.

Open a file

Linux


							$ xdg-open file_name.txt

$ xdg-open followed by a name + extension will open that file using the default editor for that kind of file on Linux.

Open a file

Windows


							$ start file_name.txt

If you’re using windows, use $ start file_name.extension

Quick Review

  • $ pwd Print the working directory
  • $ ls List directories
  • $ cd folder_name Change to a named folder
  • $ clear Clear the view
  • $ mkdir folder_name Creates a new folder
  • $ touch file_name Creates a new file
  • $ open file_name Open the named file

A brief explanation about

Git is a distributed versioning control system.

Distributed system

Git Server

In a distributed version control system you can work in a bunch of iteration locally, and sync all at once, whenever you have access to the server.

Not distributed systems demand connection to the serve in order to generate history. It means that if you don't have network connection, you're unable to send changes to the code base.

Git is a diary

Git register changes in the project over time.

  • It stores what, who, when and why a change was made based in the information of each commit.
  • It allows you to review those changes and provide mechanisms to undo them if you need.

It's also a time machine

When something breaks git allow us to go back to a previous state

Git is part of our deployment and it allow us to rollback fast in case there’s any problems.
